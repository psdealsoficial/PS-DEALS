PS Deals Blueprint v4.0

Documento maestro para migrar PS Deals v3.9.1 LTS desde localStorage hacia SQLite con una arquitectura segura, gradual y reversible.

1. Resumen ejecutivo

PS Deals v3.9.1 ya cuenta con los módulos operativos principales. v4.0 reemplazará la persistencia del navegador por SQLite sin reescribir la interfaz.

Decisión recomendada: conservar HTML/CSS/JavaScript y agregar una API local Node.js + Express + SQLite. El navegador no abrirá directamente la base.

2. Arquitectura objetivo

Presentación

HTML, CSS, Bootstrap y JavaScript existentes. No accede directamente a SQLite.

Cliente de datos

Módulo js/api-client.js. Sustituye llamadas directas a localStorage por solicitudes HTTP.

API local

Node.js + Express. Valida datos, aplica permisos y ejecuta transacciones.

Dominio/servicios

Reglas de ventas, inventario, cuentas por cobrar, apartados, devoluciones y auditoría.

Persistencia

SQLite con claves foráneas, índices, restricciones y migraciones.

Respaldo

Copias consistentes de psdeals.db, exportación JSON y registro de restauraciones.

5. Migración

  1. 0. Congelación: Commit, ZIP del código y respaldo JSON de datos.
  2. 1. Inventario: Detectar claves localStorage, estructura y conteos.
  3. 2. Base vacía: Crear SQLite mediante migraciones SQL versionadas.
  4. 3. Importación: Transformar productos, clientes, ventas y demás colecciones.
  5. 4. Conciliación: Comparar conteos, totales de ventas, saldos y existencias.
  6. 5. Prueba paralela: Usar copia de datos; conservar localStorage como solo lectura.
  7. 6. Corte: Activar API/SQLite y bloquear escrituras antiguas.
  8. 7. Observación: Revisar errores, respaldos y rendimiento durante varios días.
  9. 8. Cierre: Eliminar modo compatibilidad solo tras aceptación.

7. Primer sprint: v4.0.0

  • API local y endpoint de salud.
  • SQLite y migración inicial.
  • Cliente de datos con modo conmutable.
  • Diagnóstico de conexión.
  • Datos de prueba; aún sin migrar ventas reales.

8. Criterios de aceptación

  • Conteos, totales, saldos y existencias conciliados.
  • Respaldo y restauración probados.
  • Permisos aplicados en servidor.
  • Cero errores críticos en Diagnóstico.
